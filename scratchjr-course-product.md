## 6. Redesigning the Curriculum Structure
My first step was to reorganize the lessons around concepts instead of only themes. I rebuilt the lesson list into a table with columns such as:   
‚Ä¢	Lesson number and name.   
‚Ä¢	Main activity / project file.   
‚Ä¢	Core programming concept (Basics, Sequence, Condition, Loop, Project Integration).   
‚Ä¢	Related questions or prompts for discussion.   
From there, I proposed a new structure that:   
‚Ä¢	Starts with **basic operations** (open app, create characters, move, run scripts).   
‚Ä¢	Moves into **sequencing** (ordering blocks so actions happen in the right order).   
‚Ä¢	Introduces **conditional logic** (reacting to events or collisions).   
‚Ä¢	Adds **loops** (repeating patterns without copying blocks).   
‚Ä¢	Ends with **integration projects** where kids combine multiple concepts in their own stories.   
Right now, the detailed design is complete up to the early units, and the later units are planned as creative projects that combine earlier blocks rather than fixed step‚Äëby‚Äëstep builds.



## 7. Motivation and Reward System (Online‚ÄëFriendly)
Because online lessons can be watched at any time, a big risk is that families ‚Äúmean to start‚Äù but never build a habit. To support this, I designed a simple points and badges system that families can run themselves.
At a high level, the system has:
‚Ä¢	**Points** for completing lesson videos, projects, and extra remixes.
‚Ä¢	**Badges** for concepts (sequence, condition, loop), consistency, and creativity.
‚Ä¢	A printable reward poster and badge sheet that parents can put on the fridge and customize with their own prizes.
The visuals and concrete rules are in the appendix section below.

### Printable Reward System Artifacts
To make the motivation system easy to run at home, I designed two printable artifacts:
